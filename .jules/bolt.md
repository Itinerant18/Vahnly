## 2026-06-13 - Avoid unnecessary sync.Mutex locks in sequential routing logic
**Learning:** In the backend routing logic (`internal/routing/graph/contraction_hierarchies.go`), `ComputeShortestPathETA` is designed to be executed sequentially per invocation. Wrapping local heap pop and push operations inside tight inner loops with a `sync.Mutex` acts as a performance anti-pattern. Because these data structures are scoped strictly to the function call's execution stack and not shared across threads, standard synchronization is pointless and creates substantial locking overhead per-node traversal.
**Action:** When inspecting tight algorithmic loops running on localized scopes (like A* or Dijkstra implementations), verify whether variables truly escape the local routine context before adding or maintaining heavy thread-safety locks. If the scope is entirely localized, aggressively remove `sync.Mutex` implementations around those variables to streamline algorithmic performance.
