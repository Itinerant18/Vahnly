## 2024-06-03 - KuhnMunkres algorithm allocation overhead
**Learning:** During large-scale batch dispatch windows, the core matching engine runs the Hungarian algorithm (`SolveKuhnMunkres`). We discovered that inner-loop array allocations (`minv` and `used`) were happening O(n) times per solver execution (where n is the dimension of the cost matrix). This caused significant memory allocation overhead (~109KB and 215 allocs per call for a 100x100 matrix), increasing garbage collection pressure and reducing concurrency throughput.
**Action:** Always inspect nested loops in frequently executed path code (especially ML/algorithmic solvers) for unnecessary allocations. Hoisting slice allocations outside the loop and clearing/reinitializing them in-place with `clear()` significantly reduced memory overhead (from 109KB to 9KB, 215 allocs to 17 allocs for n=100) and reduced execution time by 30-40%.
